# CleanFeedVK — контекст проекта

iOS-клиент ВКонтакте без рекламы и алгоритмической ленты. Цель — App Store релиз.
Стек: iOS 16+, SwiftUI, MVVM, VK API (OAuth через Kate Mobile token).

## Как работать

1. Прочитай **tasks.json** — выбери задачу с наивысшим priority и status=pending (проверь что dependencies=done)
2. Прочитай **PROJECT_STATUS.md** — текущие блокеры и активный трек
3. **PRD-CleanFeedVK-2026-04-04.md** — глобальный план, читать при стратегических вопросах

## Правила

- Работать над одной задачей за сессию
- Не трогать код вне текущей задачи

## Обновление файлов и коммиты

Обновлять файлы **в конце каждой сессии** (не только в конце трека), только по команде пользователя:

1. `tasks.json` — поставить status=done для завершённых задач
2. `PROJECT_STATUS.md` — убрать закрытые блокеры, обновить активный трек, зафиксировать «где остановились»
3. `PROJECT_STATUS_ARCHIVE.md` — перенести завершённые задачи из PROJECT_STATUS
4. `progress.md` — добавить запись что сделано
5. `git commit` — один коммит на трек, делает **пользователь самостоятельно**

Не создавать промежуточных коммитов и не обновлять статусные файлы в середине работы.
В конце сессии напомнить пользователю обновить файлы, даже если трек не завершён.

## Справочники (читать по необходимости)

- `TECHNICAL_NOTES.md` — технические решения и архитектурные детали
- `MESSAGES_SPEC.md` — спецификация сообщений и вложений
- `PROJECT_STATUS_ARCHIVE.md` — архив выполненных задач
