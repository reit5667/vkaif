# Промпт для следующей сессии

Проект: **CleanFeedVK** (iOS 17+, Swift 6, SwiftUI, MVVM).  
Актуальный статус и открытые задачи — **PROJECT_STATUS.md**.  
Архив завершённых фич — **PROJECT_STATUS_ARCHIVE.md** (при необходимости справки по коду; в контекст по умолчанию не тащить).

---

## Вводные с прошлой сессии (замечания)

1. **Фото не добавляется в сохранённые** — при нажатии «Добавить в сохранённые» результат не получен, фото не отображается в альбоме. Нужно починить: разобрать вызов photos.copy (параметры, scope, access_key?), ответ API и то, как/где должно отображаться сохранённое фото.

2. **Плеер видео**
   - **«Возобновить» не работает** — с паузы снять не получается.
   - **«Повторить» работает как «восстановить»** — сбрасывает видео в состояние, где снова можно поставить на паузу и продолжить просмотр, рекомендации убираются — это хорошо.
   - **Рекомендации** — мы их оставили, но они не выводятся: пустой блок и стрелки. Лучше либо чтобы рекомендации отображались, либо убрать пустое место.
   - Нужно **подшаманить кнопку «Повторить»**: сейчас она по сути «возврат к просмотру с места» (и убирает рекомендации). Либо переименовать под это поведение, либо добавить отдельно «с начала» и т.п.

3. **Фото профиля** — по-прежнему открывается в миниатюре (в шапке и при открытии на полный экран). Задача остаётся в бэклоге (PROJECT_STATUS).

## Задачи для следующей сессии (по PROJECT_STATUS)

**В приоритете (блокеры):**
1. Исправить **«Добавить в сохранённые»** — фото должно реально попадать в альбом и/или показывать успех/ошибку.
2. **Плеер:** починить или заменить «Возобновить»; доработать «Повторить» (поведение/название); рекомендации — либо показывать, либо убрать пустой блок со стрелками.
3. **Фото профиля** — при открытии аватара показывать нормальное фото (не миниатюру), см. PROJECT_STATUS.

**Дорожная карта:** Сообщения (messages.get, messages.getConversations) — в последнюю очередь.

**Бэклог:** репосты, превью фото в сетке (по желанию).

## Технические ориентиры

- **VideoPlayerView:** кнопки «Возобновить» и «Повторить» в шапке; videoBridgeScript, keepVideoVisibleScript, hideOverlaysScript; requestPlay через postMessage (может не доходить из cross-origin iframe).
- **FullScreenPhotoGalleryView:** photoIdsForSaving, onAddToSaved; при ошибке сохранения показывается «Не удалось сохранить»; в ленте/группе/профиле передаётся callback addPhotoToSaved → vkApi.photosCopy.
- **VKApiService:** photosCopy(ownerId, photoId, accessKey?), остальные методы — в PROJECT_STATUS_ARCHIVE при необходимости.
