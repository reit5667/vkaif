## 1. Обзор и цели

**CleanFeedVK** — нативный iOS-клиент ВКонтакте без рекламы, алгоритмических рекомендаций и навязчивых функций. Вдохновлён Kate Mobile (Android) и классическим интерфейсом ВКонтакте эпохи 2012–2016 гг.

### Цели продукта
- Дать пользователям ВКонтакте инструмент, в котором лента состоит **только** из их собственных подписок и сообществ
- Убрать рекламу, Stories, клипы-рекомендации и прочий нерелевантный контент
- Обеспечить быстрый, лёгкий, предсказуемый опыт использования
- Выйти в **App Store** с рабочим продуктом через 3–4 месяца

### Цель версии 1.0 (App Store)
Стабильное, полнофункциональное приложение с основными разделами: Лента, Сообщения, Профиль, Видео/Клипы, Музыка (базовая). Freemium-модель монетизации.

---

## 2. Целевая аудитория

**Основной сегмент:** Пользователи ВКонтакте, которые годами формировали подписки и сообщества, чтобы получить "свою" ленту — и теперь официальное приложение ломает этот опыт рекламой и алгоритмами.

**Характеристики:**
- Активные пользователи ВКонтакте (не собираются уходить из сети)
- Ценят контроль над контентом и чистый интерфейс
- Знакомы с Kate Mobile или помнят старый ВКонтакте
- iOS-пользователи (iPhone, возможно iPad в будущем)
- Возраст: преимущественно 20–40 лет

**Размер аудитории:** Нишевый продукт, но ниша измеримая — по аналогии с Kate Mobile на Android (миллионы пользователей).

### 3.2 Функции для версии 1.0

#### Лента
- Хронологическая лента только из подписок (без рекламы, promo, marked_as_ads)
- Посты с текстом, фото, видео, ссылками, опросами, документами
- Лайки, комментарии, репосты
- Удаление и закрепление своих постов
- Время поста (добавить к дате)
- **Acceptance criteria:** В ленте нет ни одного рекламного поста; лента загружается менее 2 секунд; бесконечная прокрутка работает без сбоев

#### Сообщения
- Список диалогов с пагинацией
- Чат: отправка текста и фото
- Вложения в входящих: фото, видео, стикеры, голосовые сообщения, репосты
- Reply на сообщение
- Удаление своих сообщений
- **Acceptance criteria:** Все типы вложений отображаются корректно; история загружается при скролле вверх; стикеры рендерятся как изображения

#### Профиль
- Своя страница: фото профиля, статус, стена, альбомы
- Профили других пользователей и страниц сообществ
- Загрузка и смена аватара
- **Acceptance criteria:** Загрузка аватара работает без ошибок; переход на профиль из ленты < 1 сек

#### Видео и Клипы
- Просмотр видео из ленты и со стены
- Отдельный раздел «Клипы» (вертикальное видео)
- Убрать рекомендации ВКонтакте по окончании видео
- **Acceptance criteria:** Видео воспроизводится без буферизации на Wi-Fi; рекомендации скрыты

#### Музыка (базовая, для v1.0)
- Список аудиозаписей пользователя (audio.get)
- Воспроизведение треков
- Фоновое воспроизведение (AVAudioSession)
- **Acceptance criteria:** Треки играют в фоне; не останавливаются при блокировке экрана

## 4. Технический стек

| Слой | Решение | Статус |
|------|---------|--------|
| Платформа | iOS 16+, SwiftUI | Текущий |
| Архитектура | MVVM + DI (NetworkService) | Текущий |
| API | VK API через OAuth 2.0 (Kate Mobile token) | Текущий |
| Хранение токена | Keychain | Текущий |
| Сетевой слой | async/await, URLSession | Текущий |
| Видео | WKWebView + bridge scripts | Текущий |
| Фото | PHPhotoLibrary, PhotosPicker | Текущий |
| Музыка | AVPlayer + AVAudioSession | Добавить |
| In-App Purchase | StoreKit 2 | Добавить |
| Аналитика | опционально: TelemetryDeck (privacy-first) | Рассмотреть |

### Конфиденциальность
- Авторизация через официальный VK OAuth (Kate Mobile client_id)
- Токен хранится только в Keychain устройства
- Никакого стороннего трекинга (исключить Firebase Analytics, Amplitude и подобные)
- App Privacy в App Store: заявить минимальный сбор данных
- Рассмотреть Privacy Manifest (обязателен для App Store с 2024)

## 5. Модель данных (ключевые сущности)

```
VKUser
  id: Int, firstName: String, lastName: String
  photo200: URL, status: String, domain: String

VKPost
  id: Int, ownerId: Int, date: Int, text: String
  attachments: [VKAttachment], likes: VKLikes
  comments: VKComments, reposts: VKReposts
  markedAsAds: Bool, isPinned: Bool

VKAttachment (enum)
  .photo(VKPhoto), .video(VKVideo), .audio(VKAudio)
  .link(VKLink), .doc(VKDoc), .poll(VKPoll)
  .sticker(VKSticker), .voiceMessage(VKVoiceMessage)

VKMessage
  id: Int, fromId: Int, peerId: Int, date: Int, text: String
  attachments: [VKAttachment], replyMessage: VKReplyMessage?
  isDeleted: Bool

VKAudio
  id: Int, ownerId: Int, artist: String, title: String
  duration: Int, url: URL?

VKConversation
  peer: VKPeer, lastMessage: VKMessage
  unreadCount: Int
```

## 6. UI-принципы

- **Чисто и быстро:** минимум декоративных элементов, максимум контента
- **Нативный iOS:** стандартные компоненты SwiftUI, системные иконки SF Symbols
- **Без Stories и клипов в ленте:** отдельные разделы для нишевого контента
- **Тёмная тема:** поддержка системной светлой/тёмной темы
- **Таббар:** Лента | Клипы | Сообщения | Музыка | Профиль (обновить при добавлении музыки)
- **Типографика:** системный шрифт (SF Pro), justified text только там где нужно — проверить межбуквенный интервал

## 7. Этапы разработки (3–4 месяца до App Store)

### Фаза 1 — Стабилизация (Апрель, ~4 недели)
**Цель:** закрыть все известные баги и блокеры

- [x] Исправить загрузку аватара (проверено вручную)
- [x] Исправить краш при скачивании фото
- [ ] Исправить удаление фото из альбома
- [x] Исправить пагинацию в сообщениях ("Загрузка старых сообщений" зависает)
- [ ] Убрать рекомендации по окончании видео
- [ ] Исправить межбуквенный интервал в постах (justified text)
- [ ] Добавить время к дате поста

**Acceptance criteria фазы:** все перечисленные баги закрыты, приложение не крашится в стандартных сценариях.

### Фаза 2 — Вложения в сообщениях (Апрель–Май, ~3 недели)
**Цель:** полноценные сообщения

- [ ] Отображение стикеров (рендер как изображение по URL)
- [ ] Отображение голосовых сообщений (AVPlayer, длительность, воспроизведение)
- [ ] Отображение репостов в сообщениях
- [ ] Отображение видео-вложений в чате
- [ ] Отображение документов (базовое — имя файла + скачать)

**Acceptance criteria фазы:** все типы вложений из VK API корректно отображаются в диалоге.

### Фаза 3 — Музыка (Май, ~3 недели)
**Цель:** базовый аудиоплеер

- [ ] audio.get — список треков пользователя
- [ ] Воспроизведение треков (AVPlayer)
- [ ] Фоновое воспроизведение (AVAudioSession + Now Playing)
- [ ] Мини-плеер (persistent bar над таббаром)
- [ ] Управление с экрана блокировки (MPRemoteCommandCenter)
- [ ] Базовый UI: список треков, текущий трек, play/pause/next/prev

**Acceptance criteria фазы:** трек играет в фоне, не останавливается при блокировке, управление с lock screen работает.

### Фаза 4 — Freemium + App Store (Июнь, ~3 недели)
**Цель:** монетизация и публикация

- [ ] StoreKit 2 — In-App Purchase (разовая покупка или подписка)
- [ ] Paywall для premium-функций (определить финальный список)
- [ ] App Icon, Launch Screen, скриншоты для App Store
- [ ] Onboarding (авторизация VK — объяснить пользователю что и зачем)
- [ ] Privacy Manifest (PrivacyInfo.xcprivacy)
- [ ] TestFlight — бета-тест с небольшой группой
- [ ] Исправление багов по результатам TestFlight
- [ ] Публикация в App Store

**Acceptance criteria фазы:** приложение прошло review App Store, доступно для скачивания.

## 8. Потенциальные проблемы и решения

| Проблема | Вероятность | Решение |
|----------|-------------|---------|
| VK API заблокирует Kate Mobile токен | Средняя | Следить за изменениями; рассмотреть регистрацию своего приложения в VK |
| App Store отклонит приложение (VK-клиент) | Низкая | Корректно оформить описание, не упоминать обход ограничений |
| Музыка — VK API аудио ограничено | Высокая | audio.get работает только для своих треков; проверить доступность URL заранее |
| Динамическое центрирование pinch (SwiftUI ограничение) | Средняя | UIViewRepresentable обёртка с UIScrollView |
| Конфиденциальность пользователей при выходе в публичный продукт | Средняя | Privacy Manifest, минимальный сбор данных, понятная политика конфиденциальности |
