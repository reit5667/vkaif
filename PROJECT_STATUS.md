## 4. Технические заметки

- **Картинки в постах:** из `attachments[].photo.sizes` берётся URL (приоритет x → m → s), отображается в `LazyVGrid` (1 фото — во всю ширину, 2 — два столбца, 3+ — до 3 столбцов).
- **Пагинация ленты:** при достижении конца списка вызывается `newsfeed.get(start_from: nextFrom)`, посты и авторы мержатся в текущие массивы.
- **Лайки/комментарии в постах:** VKPost.likes (count), VKPost.comments (count); под постом — строка с иконкой сердца и пузыря + число (если > 0).
- **Переход из ленты на группу/профиль:** тап по автору поста (имя/аватар) → FeedDestination.group(id) или .user(id) → GroupWallView (groups.getById + wall.get) или ProfileView(userId). В ячейке поста передаётся authService и feedDestination; на стене группы — без навигации из заголовка.
- **Профиль:** `users.get` с fields photo_200, photo_400, photo_max, photo_max_orig. Fullscreen аватара: лучший из доступных (качество зависит от VK). Вкладки: **разбор ошибки VK** — при ответе `{ "error": { "error_code", "error_msg" } }` показывается текст ошибки (VKApiError.apiError → LocalizedError). friends.get extended=1, photos.getAlbums need_covers=1. В VKApiService для getPhotosAlbums, getFriends, getGroups — логирование вызова и результата (count/items). Во вкладках — pull-to-refresh (async load + refreshable).
- **OAuth scope:** `wall,offline,friends,photos,groups` — после изменения scope нужен **перелогин** (Выйти → Войти), чтобы VK запросил новые права и вкладки Фото/Друзья/Группы получили данные.
- **Фото в ленте:** `displayURL` использует приоритет размеров x,m,s,w,z,y,r,q,p,o → первый доступный; при ошибке загрузки в ячейке показывается иконка. **Часть картинок не подгружается** — открытая проблема (формат sizes/приватность/сеть).
