## 4. Технические заметки

- **Картинки в постах:** из `attachments[].photo.sizes` берётся URL (приоритет x → m → s), отображается в `LazyVGrid` (1 фото — во всю ширину, 2 — два столбца, 3+ — до 3 столбцов).
- **Пагинация ленты:** при достижении конца списка вызывается `newsfeed.get(start_from: nextFrom)`, посты и авторы мержатся в текущие массивы.
- **Лайки/комментарии в постах:** Под постом: лайк (toggle likes.add/likes.delete, overrides счётчика), комментарии (счётчик, тап → PostCommentsView). PostCommentsView: wall.getComments, пагинация 5, «Добавить комментарий» (корневой), ответ (wall.createComment, reply_to_comment), тап по имени автора → ProfileViewWrapper (programmatic authorDestination), лайк/снять лайк комментария, thread (VKComment.thread), «Подгрузить ещё» скрывается при 0 (noMoreTopLevel). Ответ wall.createComment: декодирование VKWallCreateCommentResponse (comment_id). Кнопка комментариев показывается на всех постах (в т.ч. 0 комментариев).
- **Переход из ленты на группу/профиль:** FeedDestination → GroupWallView или ProfileViewWrapper(userId). Таб «Профиль»: ProfileView(authService, viewModel: profileViewModel) с одним @StateObject profileViewModel в ContentView.
- **Профиль:** users.get, header ок. Вкладки Фото/Друзья/Группы: ProfileViewModel.loadAlbums(ownerId:), loadFriends, loadGroups; данные приходят (логи ok), UI списков пустой — не исправлено. ProfileView принимает только @ObservedObject viewModel; для друга — ProfileViewWrapper(authService, userId).
- **OAuth scope:** `wall,offline,friends,photos,groups` — после изменения scope нужен **перелогин** (Выйти → Войти), чтобы VK запросил новые права и вкладки Фото/Друзья/Группы получили данные.
- **Фото в ленте:** `displayURL` использует приоритет размеров x,m,s,w,z,y,r,q,p,o → первый доступный; при ошибке загрузки в ячейке показывается иконка. **Часть картинок не подгружается** — открытая проблема (формат sizes/приватность/сеть).
